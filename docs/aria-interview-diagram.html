⬡

Interfaces

CLI, Electron desktop app, and MCP server — three independent entry points sharing one SQLite store

⚙

OODA Loop

Cognitive core — Observe, Orient, Decide, Act. Every task is classified and planned before any model is called

⇄

ModelRouter

Routes to Claude / Codex / Gemini / Ollama based on configurable quality, balanced, or budget profiles with health-checked fallback

◈

Blackboard

Multi-layer memory: current task → decisions → constraints → per-task experiences → global patterns. Survives daemon restarts via SQLite

⚡

Safety Guardrails

Guard hook pattern-matches 15+ destructive commands and 8 secret token patterns. Plan-gate blocks edits until a plan is approved

◎

Observability

Every event persisted to SQLite. Auto-log hook generates structured daily_log entries with duration and outcome per task
